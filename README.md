# 🔥 readme-score

> MCP server that roasts your README. Scores 7 quality dimensions with savage-but-actionable commentary. Plug into Claude Code, Antigravity, or any MCP-compatible agent.

## What is this?

`readme-score` is a lightweight MCP (Model Context Protocol) server that analyzes README files like a grumpy senior dev who's seen too many `# TODO: write docs` commits. It scores your README across **7 quality dimensions**, delivers a roast report with personality, and gives you concrete fixes.

**No API keys required.** The server fetches READMEs and provides scoring instructions — your agent's own LLM does the analysis.

### Scoring Dimensions

| Dimension | Weight | What gets checked |
|---|---|---|
| **Purpose** | 20% | Does the first paragraph actually explain what the project does? |
| **Installation** | 18% | Are there real install steps, not just "figure it out"? |
| **Usage & Examples** | 18% | Actual runnable code snippets, not pseudocode dreams |
| **Documentation Depth** | 15% | API docs, config options, architecture notes |
| **Badges & Metadata** | 10% | CI status, version badges — signals of a maintained project |
| **Contributing Guide** | 10% | PR process, issue templates, or at least a mention |
| **License** | 9% | Is the license stated? Bonus for SPDX identifiers |

### Grade Scale

| Score | Grade | Meaning |
|---|---|---|
| 90–100 | **S** | Exceptional — you'd star this repo just for the README |
| 75–89 | **A** | Solid — gets the job done with minor gaps |
| 60–74 | **B** | Decent — some sections feel rushed |
| 45–59 | **C** | Mediocre — the developer clearly had places to be |
| 30–44 | **D** | Sad — like a CV with just a name on it |
| 0–29 | **F** | A war crime against documentation |

## MCP Tools

### `readme_roast`

Score and roast a single README.

| Parameter | Type | Required | Description |
|---|---|---|---|
| `source` | string | Yes | GitHub repo URL (`https://github.com/owner/repo`) or raw README text |
| `response_format` | string | No | `markdown` (default) or `json` |
| `roast_intensity` | string | No | `mild`, `medium` (default), or `savage` |

### `readme_compare`

Compare two READMEs head-to-head and declare a winner.

| Parameter | Type | Required | Description |
|---|---|---|---|
| `source_a` | string | Yes | GitHub URL or raw README text for project A |
| `source_b` | string | Yes | GitHub URL or raw README text for project B |
| `response_format` | string | No | `markdown` (default) or `json` |

## Usage Examples

### Roast a GitHub repo
```
readme_roast(source="https://github.com/fastapi/fastapi")
```

### Roast raw text with savage intensity
```
readme_roast(source="# My Project\nIt does stuff.", roast_intensity="savage")
```

### Compare two repos
```
readme_compare(
    source_a="https://github.com/pallets/flask",
    source_b="https://github.com/django/django"
)
```

### Get JSON output for programmatic use
```
readme_roast(source="https://github.com/owner/repo", response_format="json")
```

## Agent Integration

### Antigravity / Claude Code

Add to your MCP config:

```json
{
  "mcpServers": {
    "readme-score": {
      "command": "python",
      "args": ["-m", "readme_score.server"],
      "cwd": "/path/to/readme_score_mcp"
    }
  }
}
```

No API keys needed — the agent's own LLM handles the scoring.

### Running directly

```bash
python -m readme_score.server
```

The server uses **stdio transport** — it reads/writes JSON-RPC over stdin/stdout, designed for subprocess-based MCP clients.

## How it works

```
Agent (Antigravity / Claude Code)
  ↓ calls MCP tool
readme-score server
  ├── Fetches README from GitHub (or accepts raw text)
  ├── Builds scoring prompt with grumpy-senior-dev persona
  └── Returns README + instructions to the agent
  ↓
Agent's own LLM scores and roasts the README
```

1. **Fetch** — If given a GitHub URL, fetches the raw README trying `main` → `master` branches and `README.md` → `readme.md` → `README.rst` filenames
2. **Prompt** — Wraps the README with a detailed scoring prompt (7 dimensions, grade scale, roast persona)
3. **Return** — Sends everything back to the agent, whose LLM produces the roast report

**Cost:** Zero extra — uses your agent's existing LLM.
